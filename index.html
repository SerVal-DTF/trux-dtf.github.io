Research Interests
  • Software Debugging (especially bug localization and program repair ),
  • Software Security (especially vulnerability detection and fixing),
  • Code Search (both free-form and semantic code-to-code),
  • and more broadly Secure and Reliable Software Engineering.
Open Positions:

SerVal-DTF is looking for motivated and excellent PhD candidates for applying to FNR research grants. Please send your resume and transcript to Dr. Tegawendé F. Bissyandé (tegawende.bissyande[at]uni.lu) if you are interested.

Latest News

"iFixR: Bug Report driven Program Repair" accepted @ESEC/FSE 2019

Our paper entitled "iFixR: Bug Report driven Program Repair" has been accepted for the 27th ACM Joint European Software Engineering Conference and Symposium on the Foundations of Software Engineering (ESEC/FSE 2019), written by Anil Koyuncu, Kui Liu, Tegawendé F. Bissyandé, Dongsun Kim, Martin Monperrus, Jacques Klein and Yves Le Traon.

"TBar: Revisiting Template-based Automated Program Repair" accepted @ISSTA 2019

Our paper on fix pattern based APR, entitled "TBar: Revisiting Template-based Automated Program Repair" has been accepted for the 28th ACM SIGSOFT International Symposium on Software Testing and Analysis (ISSTA 2019), written by Kui Liu, Anil Koyuncu, Dongsun Kim, and Tegawendé F. Bissyandé.

"Mining Fix Patterns for FindBugs Violations" accepted @IEEE TSE

Our paper on fix pattern mining, entitled "Mining Fix Patterns for FindBugs Violations" has been accepted for IEEE Transactions on Software Engineering (TSE 2018), written by Kui Liu, Dongsun Kim, Tegawendé F. Bissyandé, Shin Yoo, and Yves Le Traon. Accepted as a Journal-first paper at the 41st International Conference on Software Engineering, Montréal, QC, Canada, May 25–31, 2019.

"You Cannot Fix What You Cannot Find" accepted @ICST-2019

Our paper on automated program repair, entitled "You Cannot Fix What You Cannot Find! An Investigation of Fault Localization Bias in Benchmarking Automated Program Repair Systems" has been accepted for the 12th IEEE International Conference on Software Testing, Verification and Validation (ICST 2019), written by Kui Liu, Anil Koyuncu, Tegawendé F. Bissyandé, Dongsun Kim, Jacques Klein and Yves Le Traon.

"Learning to Spot and Refactor Inconsistent Method Names" accepted @ICSE-2019

Our paper on debugging method names, entitled "Learning to Spot and Refactor Inconsistent Method Names" has been accepted for the 41th ACM/IEEE International Conference on Software Engineering (ICSE 2019), written by Kui Liu, Dongsun Kim, Tegawendé F. Bissyandé, Taeyoung Kim, Kisub Kim, Anil Koyuncu, Suntae Kim, and Yves Le Traon.
